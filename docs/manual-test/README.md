# 手動テストケース

このディレクトリには、SwimHubのクリティカルユーザージャーニーを検証するための手動テストケースを格納します。

## ディレクトリ構成

```
manual-test/
├── README.md                    # このファイル
├── 01-authentication.md         # 認証関連のテストケース
├── 02-personal-practice.md      # 個人練習記録のテストケース
├── 03-personal-competition.md   # 個人大会記録のテストケース
├── 04-dashboard.md              # ダッシュボードのテストケース
├── 05-team-management.md        # チーム管理のテストケース
├── 06-team-practice.md          # チーム練習管理のテストケース
└── 07-team-competition.md       # チーム大会管理のテストケース
```

## テストケースの形式

各テストケースファイルは以下の形式に従います：

- **テストID**: 一意の識別子
- **テスト名**: テストの目的を明確に示す名前
- **前提条件**: テスト実行前に必要な状態
- **テスト手順**: ステップバイステップの操作手順
- **期待結果**: 各ステップで期待される結果
- **実際の結果**: テスト実行時の実際の結果（記入欄）
- **ステータス**: PASS / FAIL / BLOCKED
- **備考**: 追加の情報やスクリーンショットの場所など

## テスト実行のガイドライン

1. **環境準備**
   - テスト環境のURLを確認
   - テスト用アカウントを準備
   - ブラウザを最新版に更新

2. **テスト実行**
   - 各テストケースを順番に実行
   - 実際の結果を記録
   - スクリーンショットを撮影（必要に応じて）

3. **結果報告**
   - 失敗したテストケースは詳細を記録
   - バグレポートを作成（必要に応じて）
   - テスト結果をまとめて報告
